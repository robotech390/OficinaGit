Introdução ao GIT

Um guia visual para os comandos básicos.

Conceitos Fundamentais

Repositório (Repository)

É o local onde o seu projeto "vive". Ele contém todos os arquivos e o histórico de alterações. Pense nele como uma pasta de projeto com superpoderes.

git init

Área de Preparação (Staging Area)

Antes de salvar permanentemente as suas alterações (commit), você as adiciona à "Área de Preparação". É um passo intermediário que permite escolher quais mudanças serão incluídas no próximo commit.

git add <arquivo>

Commit

Um "commit" é um registro permanente das suas alterações no repositório. Cada commit tem uma mensagem descritiva que explica as mudanças realizadas.

git commit -m "Sua mensagem descritiva aqui"

Branch

Um "branch" (ramificação) é uma linha do tempo independente de commits. Permite que você trabalhe em novas funcionalidades ou correções sem afetar a linha de desenvolvimento principal (geralmente chamada de "main" ou "master").

git branch <nome-do-branch>
git checkout <nome-do-branch>

Merge

O "merge" (mesclagem) é o ato de unir as alterações de um branch em outro. É como você integra as novas funcionalidades desenvolvidas em um branch de volta à linha principal.

git merge <nome-do-branch>

Adicionando credenciais globais

Nós podemos presetar os usuarios principais do git para ele sempre saber que somos nós que estamos enviando o commit

git config --global  user.name "<seu nome no github>" 
git config --global  user.email "<seu email no github>"

Para não precisarmos colocar nosso token nós ativamos o manager de credencial

git config --global  credencial.helper manager

Fluxo de Trabalho Básico

1

Modifique seus arquivos no seu diretório de trabalho.

2

Adicione as alterações desejadas à Área de Preparação (git add).

3

Faça o commit das alterações para o seu repositório local (git commit).

4

Envie suas alterações para o repositório remoto (git push).

Feito com ❤️ para a sua aula de GIT.
